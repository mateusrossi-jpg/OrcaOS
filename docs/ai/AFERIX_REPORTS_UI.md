# AFERIX Reports UI

Goal: refine only the Reports screen visual structure.

Do not create code yet.
Do not add new features.
Do not create a heavy dashboard.
Do not duplicate budgets or closures.

## Screen order

1. Screen header
2. Main summary card
3. Category selector
4. Selected category content

## Main summary card

The first visible element after the header should be one strong hero card.

Recommended focus:
Planned vs Actual.

The card should be large, readable, premium, and comfortable on iPhone.

## Category selector

Place category selector below the hero card.

Suggested categories:
- Financeiro
- Clientes
- Servicos
- Desempenho

Use chips, segmented control, or simple premium blocks.

## Category content

Show only the selected category content.
Avoid many cards and charts at once.
Keep it clean, mobile-first, and readable.

## Visual style

Follow Aferix identity:
- dark premium
- graphite/black background
- dark cards
- yellow accent
- clean spacing
- strong hierarchy
- no teal
- no desktop ERP look

## Mobile rules

- no horizontal overflow
- no desktop grids on iPhone
- large readable cards
- comfortable spacing
- simple charts only

## Expected output

Generate UX proposal for:
- visual architecture
- hero card
- category selector
- category examples
- mobile-first behavior
- incremental implementation plan

Do not implement until real files and current props are validated.
